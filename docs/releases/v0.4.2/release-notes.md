# v0.4.2 Release Notes

发布日期：2026-03-27

## 本版本重点

- 将“打开路径”能力统一为 **打开Skills存储路径**，并放到系统原生菜单中。
- 顶部应用菜单与托盘菜单支持中英文文案，且可随应用内语言切换实时生效。
- 去除页面 Header 中重复入口，减少界面干扰。
- 修复部分环境下打开路径时出现的 `Not allowed to open path` 问题。
- 修复 `ui-ux-pro-max` 这类 symlink 仓库的安装异常，确保仓库内链接内容可正确物化安装。

## 用户可见变更

### 1) 菜单入口优化

- 顶部应用菜单新增：`功能 -> 打开Skills存储路径`
- 托盘菜单新增：`打开Skills存储路径`

### 2) 文案统一

- `快捷操作` -> `功能`
- `打开中央仓库` -> `打开Skills存储路径`

### 3) 语言联动

- 在应用内切换中英文后，顶部菜单与托盘菜单无需重启即可同步更新文案。

### 4) 页面入口调整

- 删除 Header 上的“中央仓库/路径”按钮，统一通过系统菜单进入。

## 技术实现摘要

- 前端新增语言切换事件上报，调用后端命令同步原生菜单语言。
- 后端重建并刷新 app menu + tray menu 文案。
- 路径打开改为后端系统命令调用（macOS `open` / Windows `explorer` / Linux `xdg-open`），提升兼容性与稳定性。
- Git 安装链路增强了 symlink 解析与物化：对 GitHub API symlink 条目与本地 clone 中的“指针文件”场景都进行安全解析，避免把应为目录/文件的内容误落成文本文件。

## 回归验证建议

- 切换中文/英文后，检查顶部应用菜单与托盘菜单文案是否即时变化。
- 从顶部应用菜单与托盘菜单分别打开 Skills 存储路径，确认可正常打开目标目录。
- 确认页面 Header 不再显示旧入口按钮。
